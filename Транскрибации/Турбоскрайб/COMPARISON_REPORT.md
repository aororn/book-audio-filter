# Сравнение сервисов транскрибации: Яндекс SpeechKit vs TurboScribe

**Дата:** 2026-01-30
**Версия:** 3.0 (ручная сверка)

## Методология

### Входные данные
- **Глава 2** (~38 минут аудио) — выбрана как самая короткая для детальной ручной сверки
- **Оригинал:** DOCX файл главы книги
- **Яндекс:** Транскрипция через Yandex SpeechKit API
- **TurboScribe:** Транскрипция через TurboScribe.ai (2 части: первые 30 мин + окончание)

### Обработка
1. Склейка частей TurboScribe с коррекцией таймкодов (+30 мин для второй части)
2. Удаление фантомов (метаданные чтеца: "глава два")
3. Конвертация в формат Яндекса для единообразного сравнения
4. Сравнение через `smart_compare.py` с выравниванием по якорям
5. Сверка по временным меткам для корректного сопоставления ошибок

### Ограничения
- **Глава 1** исключена из сравнения — TurboScribe обрезал окончание (~30% текста отсутствует)
- Глава 3 не анализировалась детально

## Типичные ошибки

### Яндекс (106 ошибок)

| Тип | Примеры |
|-----|---------|
| Имена собственные | комтуру→контуру, кунг→конга, нулара→мулла |
| Пропуски слов | глава→—, и→—, а→— |
| Замены | ветров→метров, выводил→уводил, заблудятся→забудется |
| Окончания | поместья→поместье, которой→которые |

### TurboScribe (96 ошибок)

| Тип | Примеры |
|-----|---------|
| Разбиение слов | вдесятеро→десятеро, мыслеречь→мысли речь |
| Окончания | седого→седова, высокомерия→высокомерие |
| Замены | скривил→скребел, калека→скаляка, кисет→кесет |
| Пропуски | либо→—, да→—, виде→— |

### Ошибки чтеца (100 ошибок — оба сервиса нашли)

| Тип | Примеры |
|-----|---------|
| Замена слов | старший→старейший, мечтатели→мечтательны, меча→клинка |
| Окончания | возможности→возможность, сообщения→сообщение |
| Пропуски | э э э→—, же→—, бы→было |
| Имена | рагедон→аргедон/рагидон, вартолом→вартовым/вортулом |

## Выводы

### 1. Качество сопоставимое

Оба сервиса имеют близкое качество распознавания:
- Яндекс: 106 собственных ошибок
- TurboScribe: 96 собственных ошибок
- Разница: ~9% в пользу TurboScribe

### 2. Оба находят ошибки чтеца

Оба сервиса успешно обнаруживают **100 ошибок чтеца** в главе 2.

### 3. Разные типы ошибок

- **Яндекс** хуже с: именами собственными (комтуру→контуру), пропусками союзов
- **TurboScribe** хуже с: разбиением слов, окончаниями (-ого→-ова)

### 4. Проблема TurboScribe с длинными файлами

Глава 1 была обрезана — TurboScribe потерял ~30% текста. Это критический недостаток для production использования.

### 5. Рекомендация

**Продолжить использовать Яндекс SpeechKit** как основной сервис:
- Стабильная работа с файлами любой длины
- Качество сопоставимо с TurboScribe
- Уже интегрирован в пайплайн
- Нет проблем с обрезкой файлов

TurboScribe может рассматриваться как резервный вариант, но требует решения проблемы с длинными файлами.
